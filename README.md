# AI-Driven Predictive Maintenance for Healthcare Equipment in Small Clinics

## Submission artefact

This package contains one reproducible implementation and the evidence generated by that implementation. All metrics in the dashboard are generated from the same model run as the CSV results; no metric is manually entered.

## Recommended verification

1. Install Python 3.12 (the reference run used Python 3.12.13).
2. Open a terminal in this folder.
3. Run `python -m pip install -r requirements.txt`.
4. Run `python 02_Python_Code/predictive_maintenance_complete.py`.
5. Confirm that `04_Execution_Outputs` and `predictive_maintenance_outputs.zip` are recreated.
6. Open `03_Working_Dashboard/predictive_maintenance_dashboard.html` in a browser.

The HTML dashboard is self-contained and does not require an internet connection.

## Contents

- `01_Notebook`: Google Colab/Jupyter version of the full workflow.
- `02_Python_Code`: standalone source code.
- `03_Working_Dashboard`: self-contained browser dashboard generated by the code.
- `04_Execution_Outputs`: datasets, predictions, metrics, reports, figures, dashboard copy, and run metadata.
- `METHODOLOGY_AND_LIMITATIONS.md`: scope, assumptions, and responsible-use statement.
- `requirements.txt`: reproducible dependency specification.

## Reproducibility

Random seed 42 is used for data generation, splitting, and model initialisation. Exact reference dependency versions are pinned in `requirements.txt`; `04_Execution_Outputs/12_run_metadata.json` records the verified environment. Image metadata may still differ across operating systems despite equivalent plots.

## Responsible use

This is an academic prototype using synthetic, rule-derived data. It is not a medical device and must not be used for real maintenance or clinical decisions without external validation using representative real-world data.
